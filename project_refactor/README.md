# Clean refactor for `project-coding`

This folder contains a refactored version of the core training and evaluation code for the
DF-MCQ and WHP-style experiments.

The goal is not to change the research question. The goal is to make each experimental
choice visible in code:

- which data are used;
- which target IDs are forgotten;
- which passages or MCQs are selected;
- which loss is optimized;
- which LoRA configuration is used;
- how MCQ entropy is computed;
- what metadata are saved with each run.

The code is intentionally split into small modules. Public comments explain what each
component does, not why a previous implementation may have been confusing.

## Main entry points

```bash
# DF-MCQ training
python scripts/train_df_mcq_clean.py \
  --model_path meta-llama/Llama-3.1-8B-Instruct \
  --train_data_path data/WHPplus/balanced_whp_mcq_train_dedup.json \
  --selected_ids config/unlearn_ids1.json \
  --lora_config config/lora_config10.json \
  --output_dir exp/clean_dfmcq_set1

# Generate WHP teacher samples with Llama
python scripts/generate_whp_teacher_samples.py \
  --model_path meta-llama/Llama-3.1-8B-Instruct \
  --names_path data/WHPplus/whp_names.json \
  --selected_ids config/unlearn_ids1.json \
  --output_dir exp/teacher_samples_set1_n20_seed1 \
  --num_samples 20 \
  --seed 1 \
  --rewrite_mode llm \
  --do_sample

# WHP training on precomputed or generated obfuscation passages
python scripts/train_whp_clean.py \
  --model_path meta-llama/Llama-3.1-8B-Instruct \
  --names_path data/WHPplus/whp_names.json \
  --obfuscate_passages exp/teacher_samples_set1_n20_seed1/obfuscate_samples.json \
  --selected_ids config/unlearn_ids1.json \
  --num_passages 20 \
  --lora_config config/lora_config10.json \
  --output_dir exp/clean_whp_set1_n20

# Evaluation
python scripts/evaluate_clean.py \
  --model_path meta-llama/Llama-3.1-8B-Instruct \
  --checkpoint exp/clean_dfmcq_set1/checkpoint.final/pytorch_model.pt \
  --lora_config config/lora_config10.json \
  --testfile data/WHPplus/whp_unlearn_testset_forget_mcq.json \
  --selected_ids config/unlearn_ids1.json \
  --names_path data/WHPplus/whp_names.json \
  --outfile exp/clean_dfmcq_set1/forget_mcq_output.json
```

## Important design choices

### 1. MCQ probabilities are normalized over the choice letters

The evaluation helper computes logits for the answer letters only, applies softmax over
those letters, and then computes entropy from the normalized choice distribution. This
matches the intended MCQ probing setup where probability mass is compared across the
available choices.

### 2. WHP data selection is deterministic and nested

For WHP experiments, the selected passage indices are generated by shuffling each
person's passage pool once with a seed and then taking the first `n` indices. This means
`n=20` is a subset of `n=50` when the same seed and source pool are used.

### 3. Teacher generation is a separate step

`scripts/generate_whp_teacher_samples.py` uses the base instruction model to generate
passages about replacement people, rewrites those passages for the target person, and
saves `obfuscate_samples.json`.  The WHP training script consumes that file like any
other precomputed passage pool.  This keeps sample generation separate from optimization
so the same generated data can be reused across LoRA and optimizer sweeps.

The teacher-generation step writes:

- `obfuscate_samples.json`: `{target_name: [passage, ...]}` used by WHP training;
- `teacher_samples_detailed.json`: source names, source passages, rewritten passages,
  and sample seeds;
- `teacher_generation_manifest.json`: input file hashes and generation settings.

### 4. Run metadata are saved automatically

Each training script writes:

- `run_config.json`: command-line arguments and LoRA settings;
- `data_manifest.json`: hashes of input files and selected samples;
- `checkpoint.final/pytorch_model.pt`: trainable parameters only, matching the original
  checkpoint style.

## Module layout

```text
src/unlearning_research/
  choice.py        # choice-token lookup, choice distributions, entropy
  data.py          # dataset classes and deterministic WHP selection
  losses.py        # DF-MCQ, retain, and causal language-modeling losses
  modeling.py      # model loading, LoRA setup, adapter toggling, checkpoint I/O
  prompts.py       # prompt builders used by training and evaluation
  teacher.py       # standalone WHP teacher-sample generation
  trainer.py       # training loops
  utils.py         # JSON, logging, seeding, hashing, metadata helpers
scripts/
  generate_whp_teacher_samples.py
  train_df_mcq_clean.py
  train_whp_clean.py
  evaluate_clean.py
  make_nested_whp_selection.py
  plot_entropy_cdf_clean.py
```

## Notes

This refactor focuses on the currently relevant experiments: DF-MCQ, WHP with
precomputed or standalone-generated obfuscation passages, MCQ/open-ended evaluation,
entropy analysis, and nested data selection. Older exploratory branches such as SelfCheck
and GRPO are intentionally left out of the clean path. They can be added back as separate
modules after the main experiments are reproducible.
